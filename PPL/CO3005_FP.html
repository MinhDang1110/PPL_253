Lập trình hàm như một mô hình lập trình

CO3005 · Lập trình hàm · Trần Ngọc Bảo Duy

Lập trình hàm là gì?

Lập trình hàm là một mô hình lập trình nhìn chương trình như quá trình đánh giá biểu thức và áp dụng hàm. Nếu lập trình mệnh lệnh nhấn mạnh vào chuỗi câu lệnh làm thay đổi trạng thái, lập trình hàm nhấn mạnh vào việc biến đổi giá trị thông qua các hàm. Thay vì hỏi "cần thay đổi biến nào, theo thứ tự nào?", lập trình hàm thường hỏi "giá trị đầu vào được biến đổi thành giá trị đầu ra như thế nào?".

Trong chương này, ta tiếp cận lập trình hàm như một mô hình ngôn ngữ lập trình, không chỉ như một phong cách viết code. Các khái niệm quan trọng gồm biểu thức (expression), áp dụng hàm (function application), hàm thuần (pure function), bất biến (immutability), hiệu ứng phụ (side effect), hàm hạng nhất (first-class function), hàm bậc cao (higher-order function), bao đóng (closure) và kết hợp hàm (function composition).

Lập trình hàm (functional programming) là mô hình lập trình trong đó tính toán chủ yếu được mô hình hóa như việc đánh giá biểu thức và áp dụng hàm. Chương trình được xây dựng bằng cách biến đổi giá trị đầu vào thành giá trị đầu ra, thay vì liên tục thay đổi trạng thái của chương trình.

Trong lập trình hàm, ta thường ưu tiên:

Ý tưởngDiễn giải
Biểu thức thay cho mệnh lệnhChương trình được cấu trúc bằng các biểu thức tạo ra giá trị
Áp dụng hàm thay cho thay đổi trạng tháiTính toán được thực hiện bằng lời gọi hàm
Dữ liệu bất biến thay cho biến thay đổiTạo giá trị mới thay vì sửa giá trị cũ
Hàm thuần thay cho mã nhiều hiệu ứng phụCùng đầu vào cho cùng đầu ra và không làm thay đổi bên ngoài
Kết hợp hàm nhỏXây dựng tính toán lớn từ các hàm nhỏ

Hình ảnh cốt lõi của lập trình hàm là:

giá trị → hàm → giá trị
Python — hàm đơn giản
def square(x):
    return x * x
print(square(4))
# Kết quả: 16

Hàm square nhận một giá trị, biến đổi nó và trả về giá trị mới. Nó không cần thay đổi biến toàn cục, không ghi ra tệp, không sửa danh sách bên ngoài. Đây là dạng hàm dễ kiểm tra, dễ hiểu và dễ kết hợp.

Một tính toán lớn có thể được tạo bằng cách lồng hoặc kết hợp các hàm nhỏ:

Python — kết hợp hàm nhỏ
def increment(x): return x + 1
def double(x): return x * 2
print(double(increment(3)))
# increment(3) → 4, double(4) → 8

Tư duy mệnh lệnh và tư duy hàm

Trong lập trình mệnh lệnh, ta thường mô tả từng bước thực hiện. Ví dụ, để bình phương từng phần tử trong danh sách, ta có thể viết:

Python — phong cách mệnh lệnh
numbers = [1, 2, 3, 4]
result = []
for x in numbers:
    result.append(x * x)
print(result)

Cách viết này nhấn mạnh các bước: tạo danh sách rỗng, duyệt từng phần tử, thêm kết quả vào danh sách. Trong phong cách hàm, ta mô tả phép biến đổi trên dữ liệu:

Python — phong cách hàm với map
numbers = [1, 2, 3, 4]
result = list(map(lambda x: x * x, numbers))
print(result)

Ở đây, map thể hiện mẫu tính toán "áp dụng một hàm cho từng phần tử". Ta không trực tiếp mô tả cách tạo danh sách kết quả từng bước; ta mô tả phép biến đổi.

Khía cạnhPhong cách mệnh lệnhPhong cách hàm
Đơn vị chínhCâu lệnhBiểu thức
Trạng tháiThường xuyên cập nhậtGiảm thiểu hoặc bất biến
LặpVòng lặpĐệ quy hoặc hàm bậc cao
Trọng tâmCách thực hiện từng bướcPhép biến đổi cần biểu diễn

Điều này không có nghĩa vòng lặp luôn xấu hoặc map luôn tốt hơn. Trong nhiều ngôn ngữ như Python, list comprehension có thể vừa rõ ràng vừa gần với tư duy hàm:

Python — list comprehension
numbers = [1, 2, 3, 4]
result = [x * x for x in numbers]
print(result)
Điểm quan trọng là lập trình hàm khuyến khích người lập trình suy nghĩ về biến đổi giá trị hơn là cập nhật trạng thái.

Tính toán như đánh giá biểu thức

Trong lập trình hàm, chương trình thường được cấu trúc như các biểu thức (expressions) tạo ra giá trị. Một biểu thức có thể là hằng số, tên biến, lời gọi hàm, biểu thức điều kiện hoặc tổ hợp của nhiều biểu thức khác. Ví dụ:

Python — đánh giá biểu thức
square = lambda x: x * x
result = square(4)
# square(4) → 4 * 4 → 16

Trong Python, biểu thức điều kiện cũng tạo ra giá trị:

Python — biểu thức điều kiện
x = -5
abs_v = x if x >= 0 else -x
print(abs_v)
# Kết quả: 5

Khác với câu lệnh if thông thường, biểu thức điều kiện trên có thể nằm bên phải phép gán vì nó trả về một giá trị. Trong tư duy hàm, một chương trình tốt thường được xây dựng bằng cách kết hợp các biểu thức nhỏ:

Python — chuỗi biến đổi
def normalize(x): return x.strip().lower()
print(normalize("  Hello  "))
# Kết quả: hello

Biểu thức x.strip().lower() là sự kết hợp nhiều phép biến đổi: bỏ khoảng trắng hai đầu, rồi chuyển về chữ thường.

Hàm như phép biến đổi

Về mặt toán học, một hàm có thể được xem là ánh xạ từ tập giá trị đầu vào sang tập giá trị đầu ra:

f : A → B

Trong lập trình, hàm cũng có thể được hiểu như một phép biến đổi giá trị. Ví dụ:

Python — hàm như ánh xạ
def square(x):
    return x * x
# square : Number → Number

Một hàm lý tưởng trong lập trình hàm thường có các đặc điểm:

Đặc điểmÝ nghĩa
Xác địnhCùng đầu vào luôn cho cùng đầu ra
Không thay đổi trạng thái ẩnKhông âm thầm sửa biến toàn cục hoặc đối tượng bên ngoài
Dễ kết hợpCó thể dùng làm thành phần trong hàm lớn hơn
Dễ kiểm traChỉ cần kiểm tra quan hệ đầu vào – đầu ra
Python — hàm thuần điển hình
def fahrenheit(c):
    return (9 / 5) * c + 32
print(fahrenheit(37))
# Kết quả: 98.6

Hàm này không phụ thuộc vào trạng thái bên ngoài. Muốn kiểm thử, chỉ cần cung cấp đầu vào và so sánh đầu ra mong đợi.

Phép tính lambda như nền tảng khái niệm

CO3005 · Lập trình hàm

Vì sao nhắc đến phép tính lambda?

Lập trình hàm có gốc rễ sâu trong ý tưởng xem tính toán là định nghĩa và áp dụng hàm. Phép tính lambda (lambda calculus) là một mô hình hình thức tối giản nắm bắt đúng tinh thần này. Nó không cần biến gán, vòng lặp, đối tượng hay bộ nhớ thay đổi; nó chỉ cần hàm và áp dụng hàm.

Trong chương này, phép tính lambda chỉ được dùng như cầu nối khái niệm, không đi sâu vào lý thuyết hình thức đầy đủ. Mục tiêu là giúp người học hiểu vì sao lập trình hàm xem hàm là trung tâm của tính toán.

Có thể hình dung mối liên hệ:

Mối liên hệ khái niệm
Phép tính lambda
  → hàm, biến, áp dụng hàm
  → lập trình hàm
  → hàm hạng nhất, hàm vô danh, hàm bậc cao, bao đóng

Nhiều khái niệm quen thuộc trong ngôn ngữ lập trình hiện đại như lambda, hàm vô danh, truyền hàm làm đối số và trả về hàm đều có liên hệ trực tiếp với phép tính lambda.

Ba khái niệm cốt lõi

Phép tính lambda được xây dựng từ ba dạng cơ bản:

Khái niệmÝ nghĩaVí dụ trong lambdaTương tự trong Python
BiếnMột tênxx
Trừu tượng hàmĐịnh nghĩa hàmλx. x + 1lambda x: x + 1
Áp dụng hàmGọi hàm(λx. x + 1) 3(lambda x: x + 1)(3)

Biểu thức λx. x + 1 có thể đọc là: hàm nhận tham số x và trả về x + 1. Trong Python:

Python — hàm lambda
inc = lambda x: x + 1
print(inc(3))
# Kết quả: 4

Ở đây, lambda x: x + 1 là một hàm không tên. Ta gán nó cho tên inc, rồi gọi inc(3).

Từ trừu tượng lambda đến hàm lambda

Một trừu tượng lambda (lambda abstraction) trong phép tính lambda tương ứng với hàm lambda (lambda function) trong nhiều ngôn ngữ lập trình. So sánh:

λx. x + 1   ↔   lambda x: x + 1

Cả hai đều mô tả cùng một ý tưởng: một hàm nhận x và trả về x + 1. Trong Python, hàm lambda có một số đặc điểm:

Đặc điểmÝ nghĩa
Không cần tên khi định nghĩaCó thể dùng trực tiếp tại chỗ
Có thể có nhiều tham sốlambda a, b: a + b
Thân hàm là một biểu thứcKhông chứa nhiều câu lệnh
Có thể truyền cho hàm khácThường dùng với map, filter, sorted
Python — lambda nhiều tham số
print((lambda a, b: a + b)(3, 4))
# Kết quả: 7

Lambda hữu ích khi cần một hàm ngắn, dùng một lần.

Python — lambda với sorted
nums = [3, 1, 2]
print(sorted(nums, key=lambda x: -x))
# Kết quả: [3, 2, 1] — sắp xếp giảm dần

Áp dụng hàm và trực giác thay thế

Áp dụng hàm (function application) nghĩa là đưa đối số vào hàm và đánh giá thân hàm. Trong trực giác đơn giản, có thể xem điều này như việc thay tham số bằng đối số trong thân hàm. Ví dụ:

(λx. x * x) 4  →  4 * 4  →  16
Python — áp dụng hàm
print((lambda x: x * x)(4))
# Kết quả: 16

Ví dụ với hai tham số:

Python — áp dụng hàm hai tham số
print((lambda a, b: a * b + 1)(3, 5))
# a = 3, b = 5 → 3 * 5 + 1 → 16

Trực giác thay thế giúp người học hiểu vì sao hàm là "động cơ tính toán" của lập trình hàm. Chương trình được thực thi bằng cách liên tục áp dụng hàm cho giá trị, tạo ra giá trị mới.

Bất biến, hàm thuần và hiệu ứng phụ

CO3005 · Lập trình hàm

Dữ liệu thay đổi được và dữ liệu bất biến

Dữ liệu thay đổi được (mutable data) là dữ liệu có thể bị thay đổi sau khi được tạo ra. Ví dụ, danh sách trong Python là đối tượng thay đổi được.

Python — dữ liệu thay đổi được
xs = [1, 2, 3]
xs.append(4)
print(xs)
# Kết quả: [1, 2, 3, 4] — danh sách bị sửa tại chỗ

Ngược lại, dữ liệu bất biến (immutable data) là dữ liệu không thể bị thay đổi sau khi được tạo ra. Ví dụ, số nguyên và chuỗi trong Python là bất biến.

Python — dữ liệu bất biến
x = 10
y = x + 1
print(x)   # 10 — không bị sửa
print(y)   # 11 — giá trị mới

Chuỗi cũng bất biến:

Python — chuỗi bất biến
s = "hello"
t = s.upper()
print(s)   # hello
print(t)   # HELLO
Lập trình hàm ưu tiên bất biến vì nó làm chương trình dễ suy luận hơn. Nếu một giá trị không thể bị sửa, ta không cần hỏi "ai đã thay đổi nó?" hoặc "nó đã bị thay đổi lúc nào?".

Hiệu ứng phụ

Hiệu ứng phụ (side effect) xảy ra khi việc đánh giá biểu thức hoặc gọi hàm làm thay đổi trạng thái chương trình, hoặc tương tác với thế giới bên ngoài.

Ví dụ về hiệu ứng phụ
x = x + 1         # sửa biến
items.append(10)  # sửa đối tượng
print(x)          # ghi ra màn hình
name = input()   # đọc từ ngoài
Hiệu ứng phụVí dụ
Sửa biến toàn cụcglobal_count += 1
Sửa đối tượng truyền vàoitems.append(x)
Ghi ra ngoàiprint(x)
Đọc từ ngoàiinput()
Gây ngoại lệraise ValueError()
Phụ thuộc trạng thái ngoàiĐọc thời gian, đọc file, đọc random
Python — hàm có hiệu ứng phụ
items = []
def add_item(x):
    items.append(x)   # sửa danh sách bên ngoài
add_item(10)
print(items)
# Kết quả: [10]

Hàm add_item không chỉ nhận đầu vào và trả kết quả. Nó làm thay đổi danh sách items bên ngoài.

Hiệu ứng phụ không phải lúc nào cũng xấu. Chương trình thực tế cần đọc dữ liệu, ghi kết quả, cập nhật cơ sở dữ liệu và giao tiếp với người dùng. Tuy nhiên, lập trình hàm khuyến khích cô lập hiệu ứng phụ và giữ phần lõi tính toán càng thuần càng tốt.

Hàm thuần

Hàm thuần (pure function) là hàm luôn tạo ra cùng một kết quả với cùng một đầu vào và không có hiệu ứng phụ.

Python — hàm thuần
def square(x):
    return x * x
# square(4) → luôn trả về 16, không sửa gì bên ngoài

Ví dụ hàm không thuần:

Python — hàm không thuần
total = 0
def add_to_total(x):
    global total
    total += x
    return total

print(add_to_total(5))   # 5
print(add_to_total(5))   # 10 — cùng đầu vào, khác kết quả!

Hàm thuần có nhiều lợi ích:

Lợi íchGiải thích
Dễ kiểm thửChỉ cần so sánh đầu vào và đầu ra
Dễ suy luậnKhông cần theo dõi trạng thái ẩn
Dễ tái sử dụngKhông phụ thuộc ngữ cảnh bên ngoài
Dễ kết hợpCó thể dùng trong pipeline hàm
Dễ song song hóaÍt rủi ro tranh chấp trạng thái

Ví dụ, thay vì sửa danh sách đầu vào:

Python — sửa tại chỗ (không thuần)
def add_one_in_place(xs):
    for i in range(len(xs)):
        xs[i] += 1

ta viết hàm tạo danh sách mới:

Python — tạo danh sách mới (thuần)
def add_one(xs):
    return [x + 1 for x in xs]
# Không thay đổi danh sách ban đầu

Tính thay thế bằng giá trị

Tính thay thế bằng giá trị (referential transparency) nghĩa là một biểu thức có thể được thay bằng giá trị của nó mà không làm thay đổi hành vi chương trình.

Nếu square là hàm thuần, thì biểu thức square(4) có thể được thay bằng 16 mà không làm thay đổi hành vi chương trình. Ngược lại, xét hàm không thuần:

Python — không có tính thay thế
counter = 0
def next_id():
    global counter
    counter += 1
    return counter

print(next_id())   # 1
print(next_id())   # 2 — không thể thay bằng hằng số!

Ta không thể thay next_id() bằng một giá trị cố định, vì mỗi lần gọi có thể cho kết quả khác.

Lợi íchÝ nghĩa
Suy luận đại sốCó thể thay biểu thức bằng giá trị tương đương
Gỡ lỗi dễ hơnÍt trạng thái ẩn
Tái cấu trúc an toàn hơnThay đổi cách viết mà ít đổi hành vi
Tối ưu hóaCompiler có thể thay lời gọi bằng kết quả nếu an toàn

Bất biến và suy luận chương trình

Trạng thái thay đổi được làm nảy sinh nhiều câu hỏi: Ai đã thay đổi giá trị này? Nó bị thay đổi khi nào? Có đối tượng nào khác đang chia sẻ dữ liệu này không? Hàm nào có thể sửa nó? Ví dụ:

Python — sửa đối tượng dùng chung
def append_zero(xs):
    xs.append(0)
    return xs
a = [1, 2, 3]
b = append_zero(a)
print(a)   # [1, 2, 3, 0] — a bị thay đổi!
print(b)   # [1, 2, 3, 0]

Danh sách a bị thay đổi, dù nhìn vào lời gọi b = append_zero(a) người đọc có thể nghĩ hàm chỉ tạo kết quả mới. Cách viết theo phong cách bất biến:

Python — tạo giá trị mới (bất biến)
def append_zero(xs):
    return xs + [0]
a = [1, 2, 3]
b = append_zero(a)
print(a)   # [1, 2, 3] — a không thay đổi
print(b)   # [1, 2, 3, 0]

Ở đây, a không bị thay đổi. Hàm tạo ra danh sách mới. Điều này giúp chương trình dễ dự đoán hơn, đặc biệt khi dữ liệu được chia sẻ giữa nhiều phần của hệ thống.

Hàm hạng nhất

Một ngôn ngữ hỗ trợ hàm hạng nhất (first-class functions) khi hàm được đối xử như các giá trị thông thường. Nghĩa là hàm có thể được gán cho biến, truyền làm đối số, trả về như kết quả, lưu trong cấu trúc dữ liệu và gọi thông qua biến.

Python — hàm là giá trị
def add(a, b):
    return a + b
f = add
print(f(3, 4))
# Kết quả: 7

Hàm cũng có thể được lưu trong danh sách:

Python — hàm trong danh sách
def inc(x): return x + 1
def double(x): return x * 2
funcs = [inc, double]
for f in funcs:
    print(f(3))
# Kết quả: 4, 6
Cách nhìn "hàm là giá trị" là nền tảng để hiểu hàm bậc cao, closure, decorator và nhiều kỹ thuật lập trình hàm.

Hàm làm đối số

Vì hàm là giá trị, một hàm có thể nhận hàm khác làm đối số. Đây là cách chương trình tham số hóa hành vi.

Python — truyền hàm làm đối số
def apply(f, x):
    return f(x)
print(apply(lambda a: a ** 2, 4))
# Kết quả: 16

Trong lời gọi apply(lambda a: a ** 2, 4), tham số f nhận hành vi "bình phương", còn x nhận giá trị 4. Một ví dụ thực tế hơn:

Python — tham số hóa điều kiện chọn
def select(xs, predicate):
    return [x for x in xs if predicate(x)]
nums = [1, 2, 3, 4, 5]
print(select(nums, lambda x: x % 2 == 0))
# Kết quả: [2, 4]

Hàm select không cố định điều kiện chọn. Điều kiện được truyền vào qua predicate. Đây là cách tăng tính tái sử dụng của mã.

Hàm làm giá trị trả về

Một hàm cũng có thể trả về một hàm khác. Điều này cho phép tạo hành vi mới một cách động.

Python — trả về hàm
def multiplier(x):
    def multiply(y):
        return x * y
    return multiply

times3 = multiplier(3)
print(times3(4))
# Kết quả: 12

Hàm multiplier(3) trả về một hàm mới. Hàm mới này ghi nhớ rằng x = 3, nên khi gọi times3(4), kết quả là 3 * 4 = 12. Có thể tạo nhiều hàm từ cùng một khuôn:

Python — nhiều hàm từ một khuôn
times2 = multiplier(2)
times5 = multiplier(5)
print(times2(10))   # 20
print(times5(10))   # 50

Đây là ví dụ quan trọng vì nó dẫn đến khái niệm closure ở phần sau.

Hàm bậc cao

Hàm bậc cao (higher-order function) là hàm nhận một hàm làm đối số, trả về một hàm, hoặc cả hai.

Python — hai dạng hàm bậc cao
# Nhận hàm làm đối số
def apply(f, x):
    return f(x)

# Trả về hàm
def multiplier(x):
    return lambda y: x * y
HàmMẫu tính toán
mapBiến đổi từng phần tử
filterChọn phần tử thỏa điều kiện
reduce hoặc foldGộp nhiều phần tử thành một giá trị
composeKết hợp nhiều hàm
DecoratorBọc hàm để mở rộng hành vi
CallbackTruyền hành vi để gọi lại sau

Hàm bậc cao giúp trừu tượng hóa các mẫu lặp thường gặp. Thay vì viết lại vòng lặp cho từng tình huống, ta đặt phần hành vi thay đổi vào một hàm.

Map, filter và reduce như các mẫu tính toán

Ba hàm bậc cao quen thuộc là map, filter và reduce.

HàmMẫuÝ nghĩa
mapBiến đổiÁp dụng hàm cho từng phần tử
filterChọn lọcGiữ các phần tử thỏa điều kiện
reduceGộpKết hợp nhiều phần tử thành một giá trị
Python — map
nums = [1, 2, 3]
print(list(map(lambda x: x * x, nums)))
# Kết quả: [1, 4, 9]
Python — filter
nums = [1, 2, 3, 4]
print(list(filter(lambda x: x % 2 == 0, nums)))
# Kết quả: [2, 4]
Python — reduce
from functools import reduce
nums = [1, 2, 3, 4]
print(reduce(lambda x, y: x + y, nums, 0))
# Kết quả: 10

Nên truyền giá trị khởi tạo cho reduce, chẳng hạn 0 trong phép cộng, để xử lý an toàn cả trường hợp danh sách rỗng.

Python — reduce với danh sách rỗng
print(reduce(lambda x, y: x + y, [], 0))
# Kết quả: 0

Kết hợp hàm

Kết hợp hàm (function composition) là kỹ thuật xây dựng hàm lớn từ các hàm nhỏ. Trong toán học, nếu có hai hàm f và g, hàm hợp được viết:

(f ∘ g)(x) = f(g(x))
Python — hàm compose
def compose(f, g):
    return lambda x: f(g(x))

def double(x): return x * 2
def increment(x): return x + 1
h = compose(double, increment)
print(h(3))
# h(3) = double(increment(3)) = double(4) = 8

Kết hợp hàm giúp tạo pipeline xử lý dữ liệu rõ ràng. Ví dụ:

Python — pipeline xử lý chuỗi
def strip_text(s): return s.strip()
def lower_text(s): return s.lower()
normalize = compose(lower_text, strip_text)
print(normalize("  Hello  "))
# Kết quả: hello

Thay vì viết một hàm lớn chứa nhiều bước, ta tách thành các hàm nhỏ, mỗi hàm có một trách nhiệm, rồi kết hợp chúng.

Bao đóng và môi trường phi cục bộ

CO3005 · Lập trình hàm

Biến tự do và môi trường phi cục bộ

Một hàm có thể sử dụng biến không được định nghĩa trong thân hàm của chính nó. Biến như vậy được gọi là biến tự do (free variable) đối với hàm đó.

Python — biến tự do
def multiplier(x):
    def multiply(y):
        return x * y   # x là biến tự do của multiply
    return multiply

Trong hàm multiply, tham số y là biến cục bộ của multiply. Nhưng x không được định nghĩa bên trong multiply. Vì vậy, x là biến tự do của multiply.

Câu hỏi là: khi multiply được gọi sau khi multiplier đã kết thúc, giá trị x lấy từ đâu?

Python — biến tự do vẫn hoạt động
times3 = multiplier(3)
print(times3(4))
# Kết quả: 12
# multiplier(3) đã kết thúc, nhưng x = 3 vẫn được giữ lại!

Hàm multiplier(3) đã chạy xong, nhưng hàm trả về vẫn cần giá trị x = 3. Python giữ lại môi trường cần thiết để multiply hoạt động. Đây chính là closure.

Bao đóng là gì?

Bao đóng (closure) là một giá trị hàm đi kèm với môi trường cần thiết để phân giải các biến tự do của nó.

closure = code + environment
Python — tạo closure
def multiplier(x):
    def multiply(y):
        return x * y
    return multiply
times3 = multiplier(3)
Thành phần của closure times3Nội dung
Codemultiply(y): return x * y
Environmentx = 3
Python — gọi closure
print(times3(4))
# Python dùng x = 3 đã giữ lại, y = 4 truyền vào → 12

Có thể tạo nhiều closure khác nhau từ cùng một hàm ngoài:

Python — nhiều closure, nhiều môi trường
times2 = multiplier(2)
times5 = multiplier(5)
print(times2(10))   # 20 — times2 nhớ x = 2
print(times5(10))   # 50 — times5 nhớ x = 5

Mỗi closure có môi trường riêng: times2 ghi nhớ x = 2, còn times5 ghi nhớ x = 5.

Vì sao chỉ có code là chưa đủ?

Giả sử ta chỉ giữ lại code của hàm bên trong:

Python — code không có môi trường
def multiply(y):
    return x * y   # x là gì?

Nếu không có môi trường, ta không biết x là gì. Sau khi lời gọi multiplier(3) kết thúc, biến cục bộ x lẽ ra không còn tồn tại trên ngăn xếp lời gọi. Nhưng hàm multiply được trả về vẫn cần x. Closure giải quyết vấn đề này bằng cách giữ lại giá trị cần thiết từ môi trường nơi hàm được tạo ra.

Python — ví dụ make_power
def make_power(n):
    def power(x):
        return x ** n
    return power

square = make_power(2)
cube   = make_power(3)
print(square(5))   # 25 — nhớ n = 2
print(cube(5))     # 125 — nhớ n = 3
Closure rất quan trọng vì nó cho phép hàm được trả về từ hàm khác mà vẫn giữ được ngữ cảnh cần thiết để hoạt động.

Decorator là gì?

Decorator là một hàm bậc cao dùng để bọc một hàm khác nhằm mở rộng hành vi của hàm đó. Decorator nhận một hàm làm đầu vào và trả về một hàm mới. Về mặt kiểu, có thể hình dung:

decorator : (A → B) → (A → B)

Nghĩa là decorator nhận một hàm từ A sang B, rồi trả về một hàm mới cũng từ A sang B, nhưng có thêm hành vi.

Python — định nghĩa decorator
def log_call(func):
    def wrapper(x):
        print("Calling...")
        return func(x)
    return wrapper

Dùng decorator:

Python — áp dụng decorator
@log_call
def square(x):
    return x * x
print(square(4))
# Calling...
# 16

Cú pháp @log_call tương đương với:

Python — tương đương không dùng @
def square(x):
    return x * x
square = log_call(square)

Decorator kết hợp hai ý tưởng: hàm bậc cao và closure. log_call là hàm bậc cao vì nó nhận hàm func và trả về hàm wrapper. wrapper là closure vì nó ghi nhớ func từ môi trường nơi nó được tạo ra.

Decorator và mở rộng hành vi

Decorator thường được dùng để thêm hành vi phụ như ghi log, đo thời gian, kiểm tra quyền hoặc ghi nhớ kết quả mà không sửa trực tiếp thân hàm gốc.

Python — decorator đo thời gian
import time

def timing(func):
    def wrapper(x):
        start = time.time()
        result = func(x)
        end = time.time()
        print("elapsed:", end - start)
        return result
    return wrapper

@timing
def slow_square(x):
    time.sleep(1)
    return x * x
print(slow_square(4))

Decorator giúp tách hành vi phụ khỏi logic chính của hàm. Hàm slow_square chỉ tập trung tính bình phương; việc đo thời gian được đặt trong decorator.

Tuy nhiên, cần dùng decorator cẩn thận. Nếu wrapper thay đổi kiểu trả về, nuốt lỗi, hoặc làm thay đổi hành vi quá nhiều, chương trình có thể khó hiểu hơn. Decorator tốt thường giữ giao diện của hàm gốc càng ổn định càng tốt.
