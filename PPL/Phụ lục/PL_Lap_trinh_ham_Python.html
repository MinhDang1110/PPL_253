Python như một ngôn ngữ đa mô hình

CO3005 · Phụ lục: Lập trình hàm trong Python · Trần Ngọc Bảo Duy

Python không phải là một ngôn ngữ lập trình hàm thuần túy. Python cho phép viết chương trình theo nhiều mô hình khác nhau: lập trình mệnh lệnh (imperative programming), lập trình hướng đối tượng (object-oriented programming) và lập trình hàm (functional programming). Khi nói "lập trình hàm trong Python", ta không có nghĩa Python bắt buộc mọi chương trình phải thuần hàm, bất biến hoặc không có hiệu ứng phụ.

Các công cụ quan trọng gồm hàm lambda (lambda function), hàm hạng nhất (first-class function), hàm bậc cao (higher-order function), map, filter, reduce, bao đóng (closure), decorator, comprehension, iterator và generator.

Xét bài toán: từ một danh sách điểm, chọn các điểm đạt, cộng thêm điểm thưởng 0.5, nhưng không vượt quá 10. Ba cách viết tương đương:

Python – Mệnh lệnh
scores = [4.5, 6.0, 7.5, 9.8]
result = []
for score in scores:
    if score >= 5.0:
        result.append(min(score + 0.5, 10.0))
print(result)  # [6.5, 8.0, 10.0]
Python – Comprehension (gần hàm)
result = [min(score + 0.5, 10.0) for score in scores if score >= 5.0]
print(result)
Python – map và filter
passed = filter(lambda score: score >= 5.0, scores)
result = list(map(lambda score: min(score + 0.5, 10.0), passed))
print(result)
Trong Python, comprehension thường được ưu tiên khi phép biến đổi đơn giản. map, filter, reduce hữu ích hơn khi ta muốn nhấn mạnh hàm như một giá trị, hoặc khi các hàm biến đổi đã được định nghĩa sẵn và có tên rõ ràng.

Hàm là giá trị trong Python

CO3005 · Phụ lục: Lập trình hàm trong Python

First-class function

Nền tảng của lập trình hàm trong Python là: hàm cũng là giá trị. Một hàm có thể được gán cho biến, truyền làm đối số, trả về từ hàm khác và lưu trong cấu trúc dữ liệu. Đây là đặc điểm của hàm hạng nhất (first-class function):

Python – Hàm là giá trị
# Gán hàm cho biến
def add(a, b): return a + b
f = add
print(f(3, 4))   # 7

# Truyền hàm làm đối số
def apply(f, x): return f(x)
print(apply(lambda x: x * x, 4))  # 16

# Trả về hàm
def make_multiplier(k):
    def multiply(x): return k * x
    return multiply
times3 = make_multiplier(3)
print(times3(4))  # 12

# Lưu hàm trong dictionary – bảng thao tác
def sub(a, b): return a - b
def mul(a, b): return a * b
ops = {"+": add, "-": sub, "*": mul}
print(ops["*"](3, 4))  # 12

Ví dụ lưu hàm trong dictionary cho thấy hàm có thể được dùng như dữ liệu để xây dựng một bộ chọn hành vi — nền tảng cho nhiều kỹ thuật như callback, command dispatch, plugin system và visitor.

Lambda

Hàm lambda trong Python là một cách tạo hàm ngắn tại chỗ. Tuy nhiên, lambda trong Python chỉ có một biểu thức, không chứa nhiều câu lệnh:

Python – lambda
print((lambda a, b: a + b)(3, 4))  # 7

# Dùng lambda với tên hàm rõ ràng hơn
def is_passed(score): return score >= 5.0
def add_bonus(score): return min(score + 0.5, 10.0)
result = list(map(add_bonus, filter(is_passed, scores)))
Điều quan trọng trong Python không phải là dùng lambda càng nhiều càng tốt, mà là dùng hàm như một giá trị khi điều đó làm chương trình rõ ràng hơn.

Biến đổi dữ liệu bằng pipeline

CO3005 · Phụ lục: Lập trình hàm trong Python

map và filter

Một trong những ứng dụng tự nhiên của lập trình hàm trong Python là xây dựng pipeline biến đổi dữ liệu. Pipeline là chuỗi các bước biến đổi, trong đó đầu ra của bước trước trở thành đầu vào của bước sau:

Python – Pipeline với map và filter
students = [
    {"id": "SV01", "name": "An", "score": 8.0},
    {"id": "SV02", "name": "Binh", "score": 4.5},
    {"id": "SV03", "name": "Chi", "score": 7.0},
]

def is_passed(student): return student["score"] >= 5.0
def to_report_row(student):
    return {"id": student["id"], "name": student["name"].upper()}

report = list(map(to_report_row, filter(is_passed, students)))
print(report)
# [{'id': 'SV01', 'name': 'AN'}, {'id': 'SV03', 'name': 'CHI'}]

# Comprehension – ngắn hơn cho trường hợp đơn giản
report = [{"id": s["id"], "name": s["name"].upper()}
          for s in students if s["score"] >= 5.0]

reduce và accumulator pattern

Python – reduce
from functools import reduce

# Tính tổng – nên dùng sum() thay thế
total = reduce(lambda acc, s: acc + s["score"], students, 0.0)
print(total)  # 19.5

# Nhóm theo đạt/không đạt (accumulator pattern)
def group_by_result(acc, student):
    key = "passed" if student["score"] >= 5.0 else "failed"
    acc[key].append(student["id"])
    return acc

summary = reduce(group_by_result, students, {"passed": [], "failed": []})
print(summary)
# {'passed': ['SV01', 'SV03'], 'failed': ['SV02']}

# Phiên bản bất biến hơn (tạo dict mới mỗi bước)
def group_by_result_pure(acc, student):
    key = "passed" if student["score"] >= 5.0 else "failed"
    return {**acc, key: acc[key] + [student["id"]]}

Bất biến và hàm thuần

CO3005 · Phụ lục: Lập trình hàm trong Python

Hàm thuần

Python không bắt buộc dữ liệu phải bất biến. Khi viết Python theo phong cách hàm, người lập trình cần chủ động hạn chế thay đổi trạng thái không cần thiết:

Python – Hàm có hiệu ứng phụ vs. hàm thuần
# Không thuần: sửa trực tiếp đầu vào
def add_bonus_bad(scores):
    for i in range(len(scores)):
        scores[i] = min(scores[i] + 0.5, 10.0)
    return scores

scores = [6.0, 7.5, 9.8]
add_bonus_bad(scores)
print(scores)  # [6.5, 8.0, 10.0] – đầu vào đã bị sửa!

# Gần thuần hơn: không sửa đầu vào
def add_bonus(scores):
    return [min(score + 0.5, 10.0) for score in scores]

scores = [6.0, 7.5, 9.8]
new_scores = add_bonus(scores)
print(scores)      # [6.0, 7.5, 9.8] – không đổi
print(new_scores)  # [6.5, 8.0, 10.0]
Đặc điểmÝ nghĩa
Cùng đầu vào, cùng đầu raKhông phụ thuộc trạng thái ẩn bên ngoài
Không hiệu ứng phụKhông sửa dữ liệu bên ngoài, không ghi/đọc ngoài nếu không cần

Tách phần thuần và có hiệu ứng phụ

Trong chương trình thực tế, không phải mọi hàm đều có thể thuần. Một cách thiết kế tốt là tách phần thuần và phần có hiệu ứng phụ:

Python – Tách phần thuần và hiệu ứng phụ
# Chưa tách tốt: lọc + biến đổi + in ra màn hình
def print_report_bad(students):
    for s in students:
        if s["score"] >= 5.0:
            print(s["name"].upper())

# Tách rõ hơn
def build_report(students):          # phần thuần
    return [s["name"].upper() for s in students if s["score"] >= 5.0]

def print_report(rows):              # phần có hiệu ứng phụ
    for row in rows: print(row)

rows = build_report(students)
print_report(rows)
build_report là phần gần với hàm thuần: nhận dữ liệu và trả về dữ liệu — dễ kiểm thử riêng lẻ. print_report là phần có hiệu ứng phụ: ghi ra màn hình.

Bao đóng và tạo hành vi có ngữ cảnh

CO3005 · Phụ lục: Lập trình hàm trong Python

Bao đóng (closure) là một hàm đi kèm với môi trường cần thiết để phân giải các biến tự do của nó. Trong Python, closure thường xuất hiện khi một hàm bên trong sử dụng biến của hàm bên ngoài và được trả về ra ngoài:

Python – Closure cơ bản
def make_multiplier(k):
    def multiply(x): return k * x
    return multiply

times3 = make_multiplier(3)
print(times3(4))  # 12 – k=3 được ghi nhớ trong closure
Python – Closure tạo hàm theo cấu hình
def make_pass_checker(threshold):
    def is_passed(student):
        return student["score"] >= threshold
    return is_passed

passed_5 = make_pass_checker(5.0)
passed_8 = make_pass_checker(8.0)
print(list(filter(passed_5, students)))
print(list(filter(passed_8, students)))
Python – Lỗi phổ biến: closure trong vòng lặp
# Lỗi: tất cả lambda cùng tham chiếu biến i
funcs = []
for i in range(3):
    funcs.append(lambda: i)
print([f() for f in funcs])  # [2, 2, 2]

# Sửa: chốt giá trị bằng tham số mặc định
funcs = []
for i in range(3):
    funcs.append(lambda i=i: i)
print([f() for f in funcs])  # [0, 1, 2]
Closure ghi nhớ biến trong môi trường, và thời điểm biến được tra cứu có thể ảnh hưởng đến kết quả. Closure không nhất thiết "copy giá trị" theo cách trực giác đơn giản.

Decorator như hàm bậc cao và closure

CO3005 · Phụ lục: Lập trình hàm trong Python

Decorator (decorator) trong Python là một ứng dụng quan trọng của hàm bậc cao và closure. Một decorator nhận một hàm và trả về một hàm mới, thường là để thêm hành vi phụ mà không sửa thân hàm gốc:

Python – Decorator ghi log
def log_call(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@log_call
def add(a, b): return a + b
print(add(3, 4))
# Calling add
# 7

# @log_call tương đương với: add = log_call(add)
Python – Decorator đo thời gian
import time
from functools import wraps

def timing(func):
    @wraps(func)   # giữ metadata của hàm gốc
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f"{func.__name__} took {end - start:.6f}s")
        return result
    return wrapper

@timing
def sum_squares(n):
    return sum(x * x for x in range(n))
print(sum_squares(100000))
Python – Decorator có tham số
def repeat(times):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            result = None
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def greet(name): print("Hello", name)
greet("An")
# Hello An
# Hello An
# Hello An
Decorator không chỉ là cú pháp @, mà là sự kết hợp của hàm bậc cao, closure và wrapping.

Iterator, generator và tính lười trong Python

CO3005 · Phụ lục: Lập trình hàm trong Python

Một điểm quan trọng khi học FP trong Python là map và filter không tạo danh sách ngay trong Python 3 — chúng trả về iterator. Điều này gần với ý tưởng đánh giá lười (lazy evaluation):

Python – Iterator và lazy evaluation
nums = [1, 2, 3]
squares = map(lambda x: x * x, nums)
print(squares)       # <map object at ...>
print(list(squares)) # [1, 4, 9]

# Generator expression – không tạo toàn bộ danh sách trung gian
nums = range(1, 1000000)
total = sum(x * x for x in nums if x % 2 == 0)
print(total)
Python – Pipeline với generator
def passed_students(students):
    return (s for s in students if s["score"] >= 5.0)

def report_rows(students):
    return ({"id": s["id"], "name": s["name"].upper()} for s in students)

rows = report_rows(passed_students(students))
print(list(rows))
Chú ý: iterator chỉ duyệt được một lần. Sau lần duyệt đầu tiên, iterator đã cạn — đây là điểm sinh viên dễ nhầm khi dùng map, filter và generator.
Python – Iterator chỉ duyệt một lần
xs = map(lambda x: x * x, [1, 2, 3])
print(list(xs))  # [1, 4, 9]
print(list(xs))  # [] – đã cạn!

Khi nào dùng phong cách hàm?

CO3005 · Phụ lục: Lập trình hàm trong Python

Phong cách hàm trong Python phù hợp khi:

  • Biến đổi danh sách, bảng ghi hoặc chuỗi dữ liệu (pipeline rõ ràng)
  • Logic xử lý có thể tách thành hàm nhỏ (dễ kiểm thử)
  • Cần tham số hóa hành vi (dùng hàm làm đối số)
  • Cần tạo hành vi theo cấu hình (dùng closure)
  • Cần thêm hành vi phụ cho hàm (dùng decorator)
Python – Ví dụ tốt
def valid(s): return s["score"] >= 5.0
def row(s): return {"id": s["id"], "name": s["name"].upper()}
report = [row(s) for s in students if valid(s)]

Không nên ép phong cách hàm khi vòng lặp rõ ràng hơn, khi cần nhiều bước có hiệu ứng phụ, hoặc khi lambda trở nên quá dài:

Python – Không nên viết quá nén
# Khó đọc
result = list(map(lambda s: {"id": s["id"], "name": s["name"].strip().upper(),
    "status": "passed" if s["score"] >= 5 else "failed"}, students))

# Viết rõ hơn
def status(score): return "passed" if score >= 5.0 else "failed"
def row(s): return {"id": s["id"], "name": s["name"].strip().upper(), "status": status(s["score"])}
result = [row(s) for s in students]
Lập trình hàm là công cụ để tăng tính rõ ràng và khả năng suy luận, không phải mục tiêu hình thức phải đạt bằng mọi giá.

Tóm tắt phụ lục

CO3005 · Phụ lục: Lập trình hàm trong Python

Python hỗ trợ nhiều kỹ thuật quan trọng của lập trình hàm:

  • Hàm là giá trị hạng nhất — có thể được truyền và trả về.
  • Lambda — tạo hàm ngắn tại chỗ.
  • map, filter, reduce — biểu diễn các mẫu biến đổi dữ liệu.
  • Closure — giúp hàm ghi nhớ môi trường tạo ra nó.
  • Decorator — mở rộng hành vi của hàm bằng cách wrapping.
  • Iterator và generator — hỗ trợ pipeline xử lý dữ liệu theo cách tiết kiệm bộ nhớ.

Tuy nhiên, Python không phải ngôn ngữ hàm thuần túy. Python không bắt buộc dữ liệu bất biến, không kiểm tra hàm có thuần hay không, không ngăn hiệu ứng phụ và không mặc định đánh giá lười toàn bộ chương trình. Vì vậy, viết lập trình hàm trong Python là một lựa chọn thiết kế — nên dùng khi chúng giúp chương trình rõ ràng, dễ kiểm thử và dễ suy luận hơn.
